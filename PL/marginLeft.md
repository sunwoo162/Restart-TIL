```
일부 정렬이 marginLeft 기반으로 맞춰져 있는데,

marginLeft:80
이 방식은 데이터 길이나 디바이스 크기에 따라 쉽게 깨질 수 있습니다.

가능하면

flex
justifyContent: "space-between"
영역 분리
같은 방식으로 레이아웃을 구성하는 것이 더 안정적입니다.
```
이는 코드리뷰로 달린 말이다. 나는 marginLeft로 맞추는줄 알았다. 
그럼 지금부터 marginLeft와 flex의 차이점에 대해서 알아보도록 하자

### marginLeft
    - 정의 : 요소의 왼쪽 테두리 바깥에 고정된 여백을 추가함.
    - 특징
        - 고정값 : px, rem, % 등 고정된 크기만큼 요소를 오른쪽으로 밈.
        - 개별 제어 : 특정 요소 하나만 이동시키고 싶을 때 사용함.
    - 한계 : 부모 요소의 크기가 변해도 여백 크기는 고정되어 있어 반응형 레이아웃에 불리함

### flex
    - 정의 : 부모요소 내에서 자식 요소들의 크기와 위치를 유연하게 배분하는 레이아웃 방식
    - 특징
        - 유연성 : 남는 공간을 비율로 나누어 가지거나, 공간이 부족할 때 축소하는 등 컨테이너 크기에 따라 아이템 크기가 자동으로 조절 됨
        - 전체 제어 : justify-content 등을 통해 아이템들을 한 번에 정렬/배치한다.
        - 반응형 : 화면 크기가 바뀌어도 자동 대응되어 반응형 웹 디자인의 핵심 기술이다.


| 특징 | margin-left | flex |
|------|------------|------|
| 속성 종류 | 박스 모델 (간격) | 레이아웃 모델 (배치) |
| 동작 방식 | 요소를 특정 수치만큼 밀어냄 | 컨테이너 공간에 맞춰 크기/위치 조절 |
| 반응형 | 불리함 (고정값) | 매우 유리함 (비율/자동) |
| 사용 범위 | 해당 요소 하나에만 영향 | 컨테이너 내의 모든 자식 요소에 영향 |
| 수치 단위 | px, em, % 등 | 비율 (flex: 1) |
