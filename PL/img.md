# img 컴포넌트
코드리뷰를 보니 에셋 사용 방식에 대해 코드리뷰가 달렸다. 
즉 img로 넣어둔 부분을 img를 컴포넌트로 만들어서 넣으라는 의미다.
왜 이렇게 해야할까
결론부터 말하자면 태그 차이 때문이다.

### img 태그
    - 색상 변경 불가능
    - 크기 조절 어려움
    - 상태 처리 제한적
    - 에셋 관리 비용 증가 가능성

이를 컴포넌트로 만들어서 태그를 변경하면

### 컴포넌트
    - 색상 변경 가능
    - 크기 조절 쉬움
    - 상태 처리 문제 없음
이렇기에 img 태그를 컴포넌트로 만들어야한다.

위에서 말한 상태 처리의 종류에는 무엇이 있을까

### 상태 처리
    - Hover : 마우스를 올렸을 때
    - Active : 클릭/활성화
    - Focus : 선택된
    - Disabled : 비활성화
이렇게 다양하게 있다.
