### 토큰
    - Access Token
        - 실제 API 요청할 때 사용
        - 유효기간 짧음 (보안)
        - 이걸 길게 설정해두면 불편함과 보안상의 문제가 있음(중간 탈취)
        - 그레서 짧게 설정하는거임 탈취 당해도 걍 재발급 해버리면 됨
        - 요약 : API를 요청할 때 사용하는 토큰 유효기간이 짧음
    - Refresh Token
        - Access Token 재발급용
        - 유효기간 김
        - 요약 : API를 요청할 때 사용하는 Access Token을 재발급 해주는 토큰 유효기간이 김

- 흐름
    1. 로그인 성공
    2. Access 토큰 + Refresh 토큰 둘 다 발급
    3. API 요청 -> Acces 토큰
    4. 만료 -> Refresh Token으로 재발급

- 토큰의 필요성
    - 매번 로그인 하기 귀찮음(저장 해준다는 소리)
    - 서버 부하 줄이기
    - 모바일 앱에 적합
    - 분산 시스템 대응
