# 하드코딩
코드리뷰에 이런 말이 달렸다. "다만 현재 구현은 UI 요소까지 png 에셋에 많이 의존하고 있고, 일부 데이터와 레이아웃도 하드코딩되어 있어
이후 유지보수나 확장 시 부담이 생길 수 있는 구조로 보입니다."
하지만 난 하드코딩이라는 단어를 처음 들어봤기에 이해할 수 없었다.
그럼 이제부터 하드코딩에 대해 알아보자

### 하드코딩이란?
    - 상수나 변수에 들어가는 값(데이터)을 코드에 직접 입력해 프로그램이 동작할 때를 말한다.

이걸보고 나는 "이게 왜 문제지?"라는 생각을 했다.
그럼 왜 문제가 될까

### 문제점
    - 코드에 데이터가 입력되어있다는 것은 곧 '고정된 값'을 의미한다. 즉 어떤 값이 오던 내가 지정해둔 값만 사용하기에 다른 값이 들어왔을때 어떨지 파악할 수 없다.
    - 즉 확장성과 유연성을 가지지 못해 코드 변경 시 오류가 발생할 확률이 높다.
    - 넣어둔 고정된 값을 변경 해야할 때 손이 많이 간다.
이와 반대된 계념으로는 소프트 코딩이 존재한다.

### 소프트코딩이란?
    - 입력받는 인자에 따라 다른 값을 출력는 것
    - 즉 그때그때 다른 값을 받아 다른 결과를 출력하는 것을 의미한다.

난 이걸 보고 의문이 들었다. 그럼 이런 문제가 있는 '하드코딩'은 어디서 사용하는거지? 지금부터 알아보겠다.

### 하드코딩이 가능한 데이터
    - 상수인 값(24시간, 원주율) 들의 경우 변수나 상수로 선언하지 않고 하드코딩을 해도 큰 문제가 없다. 정해져있기 때문

지금 당장 값을 바꾸자~
