### PNPM
```
PNPM이란?
    - performatnt NPM
    - 성능과 효율성을 강조한 패키지 매니저
    - NPM과 Varn의 대안

NPM과 PNPM의 차이점
    1. 패키지 저장 방식
        - npm : 각 프로젝트 node_modules 폴더에 모든 패키지를 복사(중복 설치 -> 디스크 낭비)
        - pnpm : 패키지 전역으로 저장 각프로젝트 node_modules 폴더에 실제 파일 대신 심볼릭 링크를 생성
    2. 성능
        - npm : package-lock.json 파일 사용
        - pnpm : 설치 속도 빠름, 디스크 용량 적음
    3. 의존성 관리
        - npm : package-lock.json 파일 사용
        - panpm : pnpm-lock.yami 파일 사용(의존성 트리의 모든 부분을 고정 -> 모든 개발자가 동일 환경 + 패키지 포장)
    4. workspace 지원
        - npm : NPM7부터 지원하긴 함 but PNPM이 더 성숙 및 효율
        - pnpm : 모노레포 프로젝트를 위한 워크스페이스 기능을 내장 -> 여러 패키지 -> 하나의 레포에서 관리 ㄱㄴ -> 종속을 효율적 공유

장점
    1. 디스크 공간 절약
        - 모든 프젝 -> 동일 패키지 복사본(디스크 공간 절약) -> 대규모 프젝, 여러 프젝 관리시 유용
    2. 빠른 설치 속도
        - 패키지 설치(NPM,Yarn)보다 빠름(심볼릭 링크를 사용해 중복된 파일 피하기 + 전역 저장소에서 패키지 가져오기)
    3. 일관된 환경 보장
        - 의존성 관리 철저히 수행 -> 개발 환경과 CI/CD 환경 간의 일관성 유지 -> 모든 개발자가 동일한 의존성을 갖춘 프젝에서 작업 보장
    4. 효율적인 모노레포 관리
        - 의존성 관리를 효율적으로 수행 + 패키지 간의 의존성을 쉽게 연결 + 중복 최소화
    5. 호환성
        - NPM 레지스트리와 완벽하게 호환 + NPM 스크립트와도 호환(NPM 프젝을 PNPM으로 쉽게 전환가능)

요약
    - 심볼릭 링크를 사용하여 패키지 관리 효율성 올리고, 디스크 사용량 줄이고, 성능 올리는 것에 중점을 둔 패키지 매니저
    - 대규모 프젝이나 모노레포 환경에서 뛰어난 성능 + 관리 효율성을 제공함
```
